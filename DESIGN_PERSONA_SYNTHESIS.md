# PFAS Design Persona Synthesis

This anchor exists because "premium startup" by itself keeps collapsing into the same polite fintech look.

The current interface is consistent, but too familiar:

- warm background
- muted teal accents
- rounded cards
- soft shadows
- quiet serif flourish

That stack is safe, but it is also generic.

## Persona Review

### 1. Brand Director

Verdict:

- coherent
- premium enough
- not memorable

Critique:

- feels like a well-executed software template
- does not yet look specific to water, infrastructure, or public evidence
- the color system is competent but predictable

Requirement:

- create a recognizable visual world, not just a polished interface

### 2. Editorial Director

Verdict:

- readable
- orderly
- too card-driven

Critique:

- the interface reads like SaaS UI before it reads like an authority product
- typography should carry more of the trust burden
- more dossier rhythm, less repeated card rhythm

Requirement:

- stronger document feeling
- better macro hierarchy
- fewer generic "pretty panel" patterns

### 3. Civic Systems Designer

Verdict:

- good decision structure
- weak domain signature

Critique:

- this should feel closer to a premium public-infrastructure console than a growth SaaS dashboard
- benchmark context, source hierarchy, and last-verified framing should feel structural

Requirement:

- more system labels
- more evidence framing
- a clearer "utility-grade interpretation" aesthetic

### 4. Product Designer

Verdict:

- high clarity
- low drama
- safe interaction model

Critique:

- the UX is honest, but the visual payoff is modest
- users should feel they entered a specialized engine, not a nice landing page with cards

Requirement:

- keep clarity
- add sharper identity
- do not trade trust for novelty

## Synthesis

Chosen direction:

`Civic Dossier`

This is a deliberate blend of:

- civic infrastructure intelligence
- editorial authority
- premium product execution

It should not look like:

- a government website
- a fintech clone
- a healthcare app clone

It should look like:

- a specialized interpretation product built on official records

## Practical Design Decisions

### Palette

Move away from default teal-led fintech.

Use:

- ink navy
- mineral blue
- brass or ochre as controlled emphasis
- parchment or archive-paper background

### Typography

Move from "friendly premium SaaS" to "technical editorial".

Use:

- IBM Plex Sans for interface and data
- Source Serif 4 for emphasis and headline modulation

### Surfaces

Reduce:

- pill-heavy softness
- blurred floating panels
- decorative gradient mood

Increase:

- dossier-like borders
- structured metadata rows
- section rules
- evidence framing

### Layout

Prefer:

- stronger asymmetry
- metadata strips
- document-style grouping
- clearer information ranks

Avoid:

- every block looking like the same rounded card

## Implementation Rule

Every page should answer this visual question:

`Does this feel like a specialized public-evidence interpretation engine?`

If the answer is "nice SaaS", it is still too generic.
